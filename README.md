# VedaVoice AI: The Human-Centric Ayurvedic Voice Consultant

## 📌 Project Overview
VedaVoice AI is a digital "Vaidya" (consultant) designed to bridge the gap between ancient Ayurvedic wisdom and modern AI accessibility. Moving beyond transactional search results, this agent uses a two-way conversational model to uncover the root of wellness concerns and provides comprehensive, explained remedies.

## ⚠️ The Problem
Traditional wellness apps often offer generic advice that lacks the depth of personalized Ayurveda:
* **Information Overload:** Users are met with static lists of "to-dos" rather than a guided experience.
* **Lack of Context:** Standard chatbots rarely understand the "Why" behind a remedy or the user's specific lifestyle factors.

## 🚀 Product & Technical Highlights

### 1. RAG-Integrated Knowledge Base
* **Verified Wisdom:** Built with a **Retrieval-Augmented Generation (RAG)** pipeline to pull data directly from a curated knowledge base.
* **Hallucination Prevention:** This architecture ensures that every suggestion is grounded rather than generic LLM outputs.

### 2. High-Fidelity Bilingual Voice (ElevenLabs)
* **Natural Conversation:** Utilizes **ElevenLabs** to deliver a warm, empathetic voice interface.
* **Accessibility:** Supports fluid, two-way dialogue in both **English and Hindi**, making holistic health accessible to those who prefer verbal interaction over typing.

### 3. "Consultant" Persona & Logic
* **Diagnostic Inquiry:** Programmed to prioritize "Diagnostic Inquiry" by asking intentional follow-up questions about diet, sleep, and symptoms before suggesting a remedy.
* **Empathetic Delivery:** Provides holistic explanations of remedies, helping users understand the "how" and "why" of their wellness journey.

### 4. Safety-First Architecture
* **Clinical Redirection:** Automatically triggers disclaimers and directs users to licensed medical professionals for conditions requiring a physical examination.

## 🛠️ Build Process
* **Vibe Coding:** Developed as the lead architect using a **vibe coding** approach. This allowed for rapid iteration on the product's conversational "soul" while managing complex API integrations and multi-modal AI workflows.
